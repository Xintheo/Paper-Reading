# 任务:根据 R 代码,为参数表按图索骥查值

你将收到两份输入:

1. **一份空白参数表**(由上一步从方法笔记生成,所有「你的选择」列为空)
2. **一段 R 脚本**(该方法对应的代码,可能包括分析代码、出图代码、或两者混合)

你的任务**只有一件**:逐行扫描参数表中的每个变量,在 R 代码里找该变量的实际值。找到就填,找不到就空着。

---

## 工作流程

### 第一步:理解表格结构

读完参数表,你看到的是一份按 `##` 标题分块的字段清单。每行有:

- 「步骤」(分组)
- 「变量」(字段名,如:待测化合物、统计方法、阈值)
- 「类型(示例)」(包含大类字母 A/B/C/D/E/F + 示例值 + 提示 + 标签)
- 「你的选择」(目前为空,需要你填)

### 第二步:逐行在代码中查值

对参数表的**每一行**,根据「变量」名 + 「类型(示例)」中的大类字母和示例值,**在 R 代码中查找对应内容**。

**查找的判断方式**(按这个顺序):

1. **代码里硬编码的数据(`data.frame`、向量 `c(...)`、`list(...)`)是否包含该字段**
    
    - 例:变量是「待测化合物」,代码里 `data.frame(Drug = c("MMC", "DOX", "ACT-D", ...))` → 找到了,值 = `MMC; DOX; ACT-D; ...`
    - 例:变量是「分类」,代码里 `Category = c("Antibiotic Agents", "Alkylating Agents", ...)` → 找到了
2. **代码里 `library(...)` / `packageVersion(...)` 调用是否对应该字段**
    
    - 例:变量是「工具」+ 示例值含 DESeq2,代码里 `library(DESeq2)` → 找到了
3. **代码里显式赋值的参数是否对应该字段**
    
    - 例:变量是「阈值」,代码里 `padj < 0.05`、`alpha = 0.05`、`min_count = 10` → 找到了
    - 例:变量是「统计方法」,代码里 `method = "BH"`、`test = "wilcox"` → 找到了
4. **图表标题/列名/标签是否揭示了该字段的语义**
    
    - 例:变量是「毒性终点」,代码里 `label = "Prediction of Respiratory Toxicity"` → 找到了,值 = `呼吸毒性(Respiratory Toxicity)`

### 第三步:填写规则

**找到对应值时**:

- 在「你的选择」列填入值,**值前加 `📎` 标记**
- 例:`📎 MMC; DOX; ACT-D; BSF; CTX; BCNU; MTX; Ara-C; GEM; CCDP; CBP; OXA; VP-16; PTX; VCR`

**找不到时**:

- 「你的选择」列**保持空白**
- 绝对不要猜、不要根据上下文推测、不要从示例值里抄过来

**值与示例冲突时**(代码里的值与「类型(示例)」给出的示例不一致):

- 仍以代码为准
- 在该行末尾追加 `⚠ 与示例冲突`

### 第四步:不要做的事

**不要新增表格里没有的字段**。即使代码里有这些,也忽略:

- 画图参数(`width = 10`、`dpi = 300`、`color = "#2166ac"`、`theme_bw()`、`element_text(...)`)
- 文件保存格式与路径(`ggsave("xxx.tiff", ...)`、`compression = "lzw"`)
- 循环变量、临时变量、中间结果变量名
- 字体、图例位置、坐标轴范围等可视化细节
- 注释行里的内容(`# 这里改成 0.01` 不算数)

判断准则很简单:**表格里没列出来的字段,就不存在**。代码再丰富也不要"扩展"表格。

**不要修改表格已有内容**:

- 不要删除任何行
- 不要修改「步骤」「变量」「类型(示例)」三列任何内容
- 只动「你的选择」这一列

### 第五步:输出前自检

逐行回看,确认:

- [ ] 标了 `📎` 的字段,在 R 代码里确实能指认到对应代码段
- [ ] 没填的字段(空白),确实是代码里找不到的(而不是漏看了)
- [ ] 没有新增任何表格里原本没有的行
- [ ] 没有修改原表的「步骤」「变量」「类型(示例)」三列

## 输出格式

**完整重新输出整张参数表**(包括所有标题分块、「📖 填表导引」块、注释),仅在「你的选择」列填入找到的值。

### 格式严格要求(必须遵守)

重新输出时,**逐字符复制原表的 markdown 格式**,只动「你的选择」列的内容。特别注意:

1. **blockquote(填表导引)与表格之间必须有一个空行**。没有空行会导致表格被渲染器吞入 blockquote,表格格式损坏。正确示例:

```
> 📖 填表导引:
> - 本步骤功能:...
> - 必填项:...
                          ← 此处必须有空行
| 步骤 | 变量 | 类型(示例) | 你的选择 |
|------|------|-------------|----------|
```

2. **表格下方的 `> ❗` 注释行与表格之间也必须有一个空行**
3. **`##` 标题与其下方的 blockquote 之间必须有一个空行**
4. **不要自行调整任何空行、缩进、标题层级**——原表怎么排,你就怎么排,区别只在「你的选择」列

在表格最开头(`# 方法参数表:xxx` 标题**之前**)追加一段「代码查值报告」:

```markdown
> 📎 **代码查值报告**
>
> - 已扫描 R 代码,共查到 **{N}** 个字段的值
> - 查到的字段:{逐个列出,格式「步骤名/变量名」,如「结构获取/待测化合物」}
> - 冲突字段:{若有 ⚠ 标记,列出;无则写「无」}
> - 代码中未匹配的内容:{代码里有数据但表格无对应字段的,简要说明;无则写「无」}
>
> 请审阅 📎 标记的值,确认无误后去掉 📎,然后继续手工填写其余空白字段。
```

在「📋 全表填表说明」的「符号含义」块加一条:

- `📎` 该值由 AI 从 R 代码中查到,请审阅确认

在「提交前自检」清单末尾加一条:

- [ ] 所有 `📎` 标记的值已核对(确认无误后去掉 `📎`,或修改为正确值)

---

## 重要原则

1. **表格字段是唯一权威**。代码里有但表格里没有的内容,不要进入输出。
2. **不要做语义扩展**。代码里有 `library(ggplot2)`,而表格里只有「工具」字段且示例是 ProTox/ADMETlab 这类预测平台,不要把 ggplot2 当成「工具」填进去——ggplot2 是出图工具,不是该步骤的核心工具。
3. **宁可不填,不要乱填**。找不到就空着,用户会手填。乱填会污染数据。
4. **不要解释推理过程**。直接输出代码查值报告 + 完整表格,不要在前后加任何说明文字。
